1.什么是多态:
    ● 对象的多种形态。
2.多态的前提:
    ● 有继承/实现关系
    ● 有父类引用指向子类对象
    ● 有方法的重写
3.多态的好处:
    ● 使用父类型作为参数，可以接收所有子类对象,
    ● 体现多态的扩展性与便利。
4.多态调用成员的特点:
    ● 调用成员变量:编译看左边，运行也看左边
    ● 调用成员方法:编译看左边，运行看右边
    ● 具体看包a2change中代码
5.多态的优势:
    ● 在多态形式下，右边对象可以实现解耦合，便于扩展和维护
    ● 定义方法的时候，使用父类型作为参数，可以接收所有子类对象,体现多态的扩展性与便利。
        例:一般jdk的工具类中的方法都会有一个Object参数方法,表示该方法可以用于接收所有类型参数。
6.多态的弊端:
    ● 不能调用子类的特有功能。
7.引用数据类型转换的方式:
    自动类型转换(向上转型)
        Person p = new Student();
    强制类型转换(向下转型)
        Student s = (Student)p;
        ● 强制类型转换可以将父类转换为真正的子类类型,从而调用子类独有的功能。
        ● 转换类型与真实对象不一致会报错。
        ● 转换时用instanceof关键字进行判断可以避免上面的问题。
