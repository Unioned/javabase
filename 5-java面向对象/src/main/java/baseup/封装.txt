1.什么是封装:
    对象代表什么,就得封装对应的数据,并提供对应的行为。
    告诉我们如何正确设计对象的属性和方法。
    原则:对象代表什么就应该封装对应的数据,并提供数据对应的行为。

2.理解封装思想的好处：
    简化开发,有什么需要处理的事情,就通过找对应的对象进行调用。
    降低学习成本,在文档中进行查找。

3.private关键字
    权限修饰符
    可以修饰成员(变量和方法)
    被private修饰的成员只能在本类中进行使用,或者通过本类中的方法进行使用。
    注：private关键字修饰数据一般用public修饰的Getter和Setter进行赋值和取值。
    例：业务实体类中的数据操作
